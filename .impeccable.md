# Impeccable — Project Design Context

This file persists design direction for the **Melody v. BCSD** citizen-journalism archive. All future sessions should read this before making visual, typographic, copy, or layout decisions.

## Design Context

### Users

**Primary:** Bibb County, Georgia parents; local and regional reporters (Macon Melody readers, Georgia Recorder, AJC, GPB); Georgia taxpayers following Open Records Act precedent; attorneys and law students researching FERPA vs. ORA tension.

**Context of use:** Long-form reading sessions, often on mobile in waiting rooms or on commutes, sometimes on desktop for research. A meaningful subset will **print filing pages** to read on paper. Many arrive cold from a single shared link (social, email, Slack) and must orient themselves quickly.

**Job to be done:** Understand what this case is about, see the primary-source filings, follow its progress, and decide for themselves whether the district or the newspaper is in the right — without a lawyer translating.

### Brand Personality

**Three words:** *Principled. Plainspoken. Meticulous.*

**Voice:** Neutral public-record voice — the site lets the docket speak. Claims are sourced to a specific filing or statute; legal terms are always defined in the [Glossary](docs/glossary.md); the author's candidate status is disclosed and kept strictly separate from the archive's content. No exclamation points, no rhetorical questions posed to the reader, no "shocking" / "outrageous" / loaded adjectives. Facts first, then a brief "why it matters" that stays factual.

**Emotional goal:** *Civic and documentary.* A visitor should feel the way they feel watching a careful PBS/Frontline piece or reading a ProPublica long-read — "somebody serious did the work, I can trust this, I'm being respected as an adult." Measured gravitas, not alarm.

### Aesthetic Direction

**Visual tone:** Editorial archive. Serif-led headlines (Georgia / Times) evoking legal filings and newspapers-of-record. Court-navy `#0b1c3d` with gold rule `#c9a23a` as the single accent — already established in the OG card and should propagate to on-site heading rules, callout borders, and link hover. Generous whitespace. Dense typography where appropriate (tables, docket entries). Light mode only (per `_config.yml color_scheme: light`).

**References (pull toward):**
- **ProPublica / NYT Investigates / Reuters Investigates** — editorial hierarchy, serif display type, primary-source link density, pull-quote restraint.
- **CourtListener / Justia / RECAP** — document-archive feel, utilitarian table layouts, footnote-heavy, primary-source-forward.
- **18F / USWDS / BritishColumbia.ca** — government-civic accessibility posture, plain-language emphasis, WCAG-first pattern library.

**Anti-references (push away from):**
- Campaign sites, PAC sites, personal political blogs — **explicitly not** a Kerry-Hatcher-for-School-Board property. No candidate photography, no "donate" framing, no first-person advocacy language in the archive.
- SaaS / startup aesthetics — no hero gradients, no emoji bullets, no 3D illustrations, no "book a demo" layouts, no dark-mode toggles for cool-factor.
- Advocacy-blog / newsletter aesthetics — no sidebar CTAs, no "subscribe to my Substack," no newsletter lightbox.

**Palette (already in use — preserve):**
- `#0b1c3d` — court navy (headings, rules, dark surfaces)
- `#081630` — deep navy (footer strip, deepest shadow)
- `#c9a23a` — gold (single accent: rules, hover states, callout borders)
- `#e6ecf5` — off-white text on navy
- Theme light background (body)
- Callout semantic colors: warning=red, note=blue, tip=green, key=yellow (defined in `_config.yml`; don't invent new ones)

**Typography (already in use — preserve):**
- **Display / headings:** Georgia, "Times New Roman", serif
- **Body:** theme default (system sans); true **16px on mobile** — do not reduce
- **h4 is NOT an uppercase eyebrow** on this site (`_sass/custom/custom.scss` overrides the theme); keep it a readable sub-heading

### Design Principles

1. **The document is the product.** Primary sources are the point. Every claim links to a filing, statute, or docket entry. Design serves the document — navigation, hierarchy, print styles, and link affordances come before decoration. If a design choice obscures a source link or the PDF behind it, reject it.

2. **Plain English, without dumbing down.** Legal terms of art are always linked to the [Glossary](docs/glossary.md) on first use in a page. Sentences are short. Subject-verb-object. No jargon without a parenthetical. The reader is an adult non-lawyer; treat them accordingly.

3. **Neutral voice, declared bias.** The archive reports; it does not advocate. The author's 2026 school-board candidacy is disclosed in the footer and on the About section — and that disclosure is the *only* place it appears. No campaign links, no candidate imagery, no first-person opinion in filing summaries or the Case Report narrative. When the record is ambiguous, say so.

4. **Accessibility as a minimum, print as a feature.** Target WCAG 2.2 AAA where reasonable (7:1 contrast for body text, 4.5:1 for large text as a floor). Respect `prefers-reduced-motion`. Mobile body stays at true 16px. Every filing page and the Case Report must have a **dedicated, tested `@media print` stylesheet** that strips nav/footer/callout decoration and renders cleanly to letter-size paper — readers will print to read.

5. **Mobile-first, table-aware.** Most traffic will be phones. Long-form legal prose gets 16px/1.5. Wide tables (docket, parties, references) get the scroll-hint gradient already established in `_sass/custom/custom.scss`. Never introduce a layout that breaks below 375px width.

6. **No future-surprise design.** No client-side JS animations, no lightboxes, no hover-only interactions, no behavior that breaks when JavaScript fails (the site is GitHub-Pages static Jekyll — keep it that way). If something can't be done in Jekyll + SCSS, it doesn't belong here.

## How to use this file

Before any visual, typographic, layout, or copy change, re-read the Design Context above. When in doubt:
- **Serif or sans?** Serif for display, theme sans for body.
- **Accent color?** Only `#c9a23a` gold. Nothing else.
- **New callout?** Use an existing semantic color from `_config.yml`. Don't invent.
- **Animation?** Only if it survives `prefers-reduced-motion: reduce`.
- **Voice drifting toward advocacy?** Pull it back to declarative reporting.
- **Considering a component framework / JS widget?** Don't. Static Jekyll only.
